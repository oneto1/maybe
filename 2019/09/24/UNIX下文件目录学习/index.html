UNIX下文件目录学习

发表于 2019-09-24

文件类型

  1. 普通文件
  2. 目录文件
  3. 块缓存文件(对文件设备提供带缓存的访问)
  4. 字符特殊文件(对设备提供不带缓存的访问)
  5. FIFO(命名管道)
  6. 套接字
  7. 符号链接

stat()

  • stat()第一个参数为文件路径，第二个参数是返回一个已经定义好的结构体
  • fstat()则是用的文件描述符代替路径
  • lstat()与stat()类似，当打开文件是一个符号链接时，只返回符号链接本身，而不是指向的文件
  • 在<sys/stat.h>中定义了判断文件类型的宏
宏 文件类型
S_ISREG (st_mode) 一般文件
S_ISDIR (st_mode) 目录
S_ISCHR (st_mode) 字符特殊文件
S_ISBLK (st_mode) 块特殊文件
S_ISLNK (st_mode) 符号连接
S_ISBLK (st_mode) FIFO
S_ISSOCK (st_mode) socket

文件访问权限

  • id 分有效id、实际id、设置id
  • 一般有效id等于实际id
  • 设置id 就是将执行该文件的进程有效id更换为文件所有者id ，在文件的st_mode值中

  • 对文件访问权限位也在st_mode内
  • 为了打开目录内的文件，必须有对目录的读和执行权限、以及对文件的相应权限

access()

  • open()是以有效id测试其访问权限
  • acces()是以实际id测试
__type 说明
F_OK 文件是否存在
R_OK 读
W_OK 写
X_OK 执行

umask()

  • __mask 即为文件权限描述符
  • umask()即屏蔽相应权限
  • shell 中

chmod()

  • node 除了基本的9个外，还可用
    S_ISUID|设置用户id
    :–:|:–:
    S_ISGID|设置组id
    S_ISVTX|保存粘着位(保存正文位)
    S_IRWXU|用户读写执行
    S_IRWXG|群组读写执行
    S_IRWXO|其他读写执行

  • 一个可执行文件的粘着位设置后，那么当它第一次被执行被终止的的时候，程序正文部分的一个副本会被保存在交换区，方便下次执行的时候快速载入内存

  • 也允许对一个目录设置粘着位，如/var/tmp、/tmp，必须对该目录有写权限且拥有该目录或文件才能删除或重命名文件，当然root是全能的

文件系统

  • 二图中有两个目录项指向同一个i节点，也就是链接数为2，存在于i节点。
  • 当链接数减少为0的时候才删除相应数据块，这种链接类型被称为硬链接
  • 而另一种链接方式是链接文件所指向的是另一个文件名字，类似于windows下的快捷方式，对于一个文件的简介指针这是符号链接
  • 一个空文件夹最小链接数为2，即. 和 ..

复杂定义读法
从变量名看起，先往右，再往左，碰到圆括号就调转阅读的方向；括号内分析完就跳出括号，还是先右后左的顺序。如此循环，直到分析完整个定义。

int (pfunc) (int);
找到变量名pfunc，先往右是圆括号，调转方向，左边是一个号，这说明pfunc是一个指针；然后跳出这个圆括号，先看右边，又遇到圆括号，这说明(*pfunc)是一个函数，所以pfunc是一个指向这类函数的指针，即函数指针，这类函数具有一个int类型的参数，返回值类型是int。

float ( (fp2) (int, int, float)) (int);
找到变量名fp2，往右看是圆括号，调转方向往左看到号，说明fp2是一个指针；跳出内层圆括号，往右看是参数列表，说明fp2是一个函数指针，接着往 左看是号，说明指向的函数返回值是指针；再跳出外层圆括号，往右看还是参数列表，说明返回的指针是一个函数指针，该函数有一个int类型的参数，返回值 类型是float。简言之，fp2是一个指向函数的指针，该函数接受三个参数(int, int和float)，且返回一个指向函数的指针，该函数接受一个整型参数并返回一个float。
